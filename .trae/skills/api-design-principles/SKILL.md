# API Design Principles

## Goal
确保接口具备一致性、可演进性、可测试性与良好开发体验。

## Invoke When
- 设计新接口或重构旧接口
- 评审 REST/GraphQL 规范
- 需要统一版本、错误码与分页策略

## Checklist
1. 明确资源模型与命名规则
2. 定义输入输出与错误契约
3. 设计版本与兼容策略
4. 提供示例请求与验证方式

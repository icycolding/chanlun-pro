# Git Workflow

## Goal
让团队在分支管理、提交质量与评审流程上保持一致高效。

## Invoke When
- 准备提交或拆分提交历史
- 发起 PR 或进行代码评审
- 协作开发出现冲突或流程混乱

## Checklist
1. 设计分支策略与命名规范
2. 规范提交粒度与提交信息
3. 明确 PR 模板与评审要点
4. 完成合并策略与回滚预案

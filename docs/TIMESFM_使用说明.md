# TimesFM 使用说明

## 1. 功能概览

当前项目中的 TimesFM 能力分为四部分：

- 实时预测层
- 历史分析中的事件级反事实预测
- 当前资产的历史预测回顾
- 可单独调用的预测接口

系统当前支持自动降级：

- 如果原生 TimesFM 运行条件未完全就绪，系统会自动切换到统计代理预测
- 页面和接口仍可正常使用

---

## 2. 页面如何使用

### 2.1 实时关注

操作方式：

- 首页选择资产
- 查看右侧实时关注面板

你会看到新增的三个相关模块：

- `TimesFM 预测层`
- `TimesFM 历史预测回顾`
- `风险经理`

其中：

- `TimesFM 预测层` 用于看当前资产未来 30 分钟与 120 分钟的方向、区间和不确定性
- `TimesFM 历史预测回顾` 用于看最近一段时间内，该资产历史事件预测的命中率和误差表现
- 该回顾模块支持：
  - `自动回顾`：切资产或实时关注刷新时自动更新
  - `手动回顾`：只有点击 `立即回顾` 才发起请求
  - `回顾周期切换`：支持 7 天 / 14 天 / 30 天
  - `频率切换`：支持 5m / 15m / 1h

### 2.2 市场总结

操作方式：

- 点击首页的“市场总结”

结果弹窗中会展示：

- 场景化路由
- 快评/深研
- TimesFM 预测层
- 风险经理

这里适合用来判断：

- 当前研究结论是否与预测层一致
- 当前风险是否主要来自高不确定性

### 2.3 历史分析

操作方式：

- 点击首页的“历史分析”
- 选择回看时长、事件阈值、频率等参数

结果弹窗中会看到：

- TimesFM 历史预测层
- 每个关键事件的事件级预测摘要
- 预测与实际表现偏差

这部分适合用来判断：

- 某次历史行情是否超预期
- 某类事件是否经常出现“预测正确但幅度不准”

## 3. 历史预测回顾怎么看

`TimesFM 历史预测回顾` 面板展示的是当前资产最近一段时间内的历史事件预测表现。

默认展示：

- 最近 14 天
- 默认使用 5m 频率
- 当前资产过去一段时间内识别出的历史价格事件

你可以在面板右上角切换：

- `自动回顾`
- `手动回顾`
- `立即回顾`
- `7天 / 14天 / 30天`
- `5m / 15m / 1h`

说明：

- 自动回顾适合持续盯盘时使用
- 手动回顾适合节省请求和只在需要时查看
- 回顾周期适合按你的复盘窗口切换
- 频率切换适合看更细或更粗的历史价格事件
- 模式、回顾周期、频率都会保存在浏览器本地，下次进入页面会沿用上次选择

主要指标包括：

- `30分钟命中率`
- `120分钟命中率`
- `30分钟平均误差`
- `平均 surprise`

### 3.1 指标含义

#### 30 分钟命中率

表示：

- 历史事件发生后，模型对未来 30 分钟方向判断是否正确

#### 120 分钟命中率

表示：

- 历史事件发生后，模型对未来 120 分钟方向判断是否正确

#### 30 分钟平均误差

表示：

- 模型预测的 30 分钟收益率与真实结果之间的平均绝对偏差

#### 平均 surprise

表示：

- 实际走势偏离模型预测的平均程度

如果 surprise 较高，通常说明：

- 这段时间事件冲击超预期
- 或当前模型对该类场景的解释力较弱

### 3.2 如何用这些指标

建议这样理解：

- 命中率高 + 误差小：预测层对当前资产较稳定
- 命中率高 + 误差大：方向有价值，但幅度不稳定
- 命中率低 + surprise 高：该资产近期可能处于异常 regime，需降权参考

## 4. 接口如何使用

### 4.1 当前资产预测接口

接口地址：

- `/api/news/timesfm/forecast`

请求方式：

- `POST`

请求示例：

```json
{
  "current_market": "fx",
  "current_code": "EURUSD",
  "frequency": "5m"
}
```

返回重点字段：

- `forecast_30m`
- `forecast_120m`
- `summary`
- `degraded`
- `degrade_message`

### 4.2 事件级预测接口

接口地址：

- `/api/news/timesfm/event_forecast`

请求方式：

- `POST`

请求示例：

```json
{
  "current_market": "fx",
  "current_code": "EURUSD",
  "frequency": "5m",
  "event_time": "2026-04-05T10:30:00",
  "actual_follow_30m_pct": -0.32,
  "actual_follow_120m_pct": -0.61
}
```

返回重点字段：

- `forecast_error_30m_pct`
- `forecast_error_120m_pct`
- `surprise_score`
- `forecast_30m`
- `forecast_120m`

### 4.3 历史预测回顾接口

接口地址：

- `/api/news/timesfm/review`

请求方式：

- `GET`

请求参数：

- `current_market`
- `current_code`
- `review_days`
- `max_items`

请求示例：

```bash
curl "http://127.0.0.1:5000/api/news/timesfm/review?current_market=fx&current_code=EURUSD&review_days=14&max_items=6"
```

返回重点字段：

- `analysis_count`
- `event_count`
- `stats.direction_hit_rate_30m`
- `stats.direction_hit_rate_120m`
- `stats.mae_30m_pct`
- `stats.mae_120m_pct`
- `stats.avg_surprise_score`
- `events`

## 5. 当前实现限制

### 5.1 历史预测回顾基于历史价格事件回测

历史预测回顾当前的实现方式是：

- 读取当前资产最近一段时间的历史价格数据
- 识别过去的关键价格事件
- 站在事件发生当时重新做 TimesFM 预测
- 将预测结果与真实后续 30 分钟、120 分钟走势对比

因此：

- 即使没有旧的历史分析存档，也能生成回顾结果
- 只要最近一段时间价格数据足够、且能识别到历史事件，就可以统计命中率和误差

### 5.2 当前可能处于降级预测模式

如果环境里没有完整的原生 TimesFM 推理能力，系统会自动降级。

这时：

- 页面仍会显示预测结果
- 接口仍会返回结果
- 但 `degraded` 字段会为 `true`

建议重点看：

- 方向
- 区间
- 不确定性

不建议把降级结果直接当成交易信号。

## 6. 推荐使用顺序

建议按以下顺序使用：

### 盘中判断

- 先看 `实时关注`
- 再看 `TimesFM 预测层`
- 再结合 `风险经理`

### 复盘判断

- 先看 `历史分析`
- 再看事件级预测与实际偏差
- 最后看 `TimesFM 历史预测回顾`

### 研究判断

- 先看 `市场总结`
- 再看研究流程块中的 `TimesFM 预测层`
- 判断研究结论与预测层是否一致

---

## 7. 一句话使用建议

把 TimesFM 当成：

- 当前研究系统的预测增强层
- 风险过滤层
- 历史事件复盘层

不要只看“预测方向”，要同时看：

- 区间
- 误差
- 不确定性
- 历史回顾表现
